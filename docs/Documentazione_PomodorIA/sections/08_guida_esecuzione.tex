\section{Guida all'esecuzione}
\label{sec:esecuzione}

Questa sezione fornisce le istruzioni operative complete per configurare ed
eseguire il progetto da zero su una nuova macchina.

\subsection{Prerequisiti}

\begin{itemize}
    \item Python 3.10 o superiore.
    \item Una copia locale del dataset \emph{PlantVillage -- Tomato}
    (ottenibile, ad esempio, tramite la libreria \texttt{kagglehub}, come
    nel progetto di training originale).
    \item Il checkpoint della rete già allenata (\texttt{CNN\_64f-k3-3blk.pth}),
    non incluso nel repository per motivi di dimensione e da copiare
    manualmente nella cartella \texttt{models/}.
\end{itemize}

\subsection{Passo 1 -- Ambiente virtuale e dipendenze}

\begin{lstlisting}[language=bash]
cd PomodorIA

# Crea un ambiente virtuale isolato
python3 -m venv venv

# Attiva l'ambiente (da ripetere ad ogni nuova sessione di terminale)
source venv/bin/activate          # macOS / Linux
# venv\Scripts\activate           # Windows

# Installa le dipendenze del progetto
pip install --upgrade pip
pip install -r requirements.txt
\end{lstlisting}

\subsection{Passo 2 -- Estrazione del test set corretto}

Come discusso nella Sezione~\ref{sec:compressione}, per ottenere misure di
accuratezza rappresentative è necessario isolare il sottoinsieme di test
(20\%) mai osservato durante il training, riproducendo lo split originale
(\texttt{seed=42}):

\begin{lstlisting}[language=bash]
python scripts/extract_test_set.py \
    --dataset-root "/percorso/al/dataset/plantvillage" \
    --output-dir "./plantvillage_testset" \
    --mode symlink
\end{lstlisting}

Il parametro \texttt{--dataset-root} deve puntare alla stessa copia del
dataset (stessi file, stesso ordine alfabetico) utilizzata durante il
training originale, condizione necessaria affinché la riproduzione dello
split casuale produca esattamente lo stesso sottoinsieme di test.
L'opzione \texttt{--mode symlink} (predefinita) crea collegamenti simbolici
anziché copie fisiche dei file, evitando di duplicare inutilmente spazio su
disco.

\subsection{Passo 3 -- Configurazione}

Verificare che il file \texttt{config.yaml} punti alla cartella del test
set appena estratta:

\begin{lstlisting}[language={}]
paths:
  dataset_root: "./plantvillage_testset"
  checkpoint: "./models/CNN_64f-k3-3blk.pth"
\end{lstlisting}

\begin{warnbox}
Se si utilizza la dashboard Streamlit, ricordare che il file
\texttt{config.yaml} viene letto una sola volta all'avvio del processo e
tenuto in cache per tutta la sua durata. Ogni modifica al file richiede
l'arresto (\texttt{CTRL+C}) e il riavvio del comando
\texttt{streamlit run}: un semplice aggiornamento della pagina nel browser
\emph{non} è sufficiente.
\end{warnbox}

\subsection{Passo 4 -- Generazione dei modelli compressi (opzionale)}

Necessario solo se si intende utilizzare la modalità \texttt{optimized} o
visualizzare i grafici di confronto nella dashboard:

\begin{lstlisting}[language=bash]
python models/compress.py
# oppure, per una sparsità di pruning diversa dal 30% predefinito:
python models/compress.py --sparsity 0.5
\end{lstlisting}

Al termine, la cartella \texttt{models/optimized/} conterrà le tre varianti
compresse descritte nella Sezione~\ref{sec:compressione}, e il file
\texttt{benchmarks/benchmark\_results.csv} verrà aggiornato con le relative
metriche.

\subsection{Passo 5 -- Avvio del sistema}

Sono disponibili due modalità di esecuzione equivalenti, descritte in
Sezione~\ref{sec:architettura}.

\subsubsection{Dashboard interattiva (consigliata per la dimostrazione)}

\begin{lstlisting}[language=bash]
streamlit run dashboard/app_streamlit.py
\end{lstlisting}

Il comando apre automaticamente il browser all'indirizzo
\url{http://localhost:8501}. Dalla barra laterale è possibile:

\begin{itemize}
    \item scegliere la modalità del modello (\emph{Full Precision} o
    \emph{Ottimizzato}) e, in quest'ultimo caso, la variante compressa
    specifica da utilizzare (Sezione~\ref{sec:compressione});
    \item forzare la fotocamera virtuale a campionare da una classe
    patologica specifica, per dimostrazioni mirate;
    \item impostare l'intervallo fra i cicli e il numero massimo di cicli;
    \item avviare, fermare e resettare la simulazione.
\end{itemize}

\subsubsection{Esecuzione da riga di comando}

\begin{lstlisting}[language=bash]
# Esecuzione base, 50 cicli
python orchestrator/main_loop.py --cycles 50 --interval 2

# Forzare una classe specifica (demo mirata)
python orchestrator/main_loop.py --class Tomato___Early_blight --cycles 20

# Selezionare esplicitamente modalità e variante del modello
python orchestrator/main_loop.py --mode optimized --variant onnx --cycles 100

# Esecuzione infinita (fino a CTRL+C), output minimo
python orchestrator/main_loop.py --cycles 0 --quiet
\end{lstlisting}

Al termine dell'esecuzione, il file di log generato in \texttt{logs/} può
essere analizzato con strumenti standard per l'analisi tabellare (ad esempio
la libreria \texttt{pandas}, oppure aperto direttamente con un foglio di
calcolo).
